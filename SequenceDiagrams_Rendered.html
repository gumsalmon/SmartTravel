PHẦN 1: WEB ADMIN (QUẢN TRỊ VIÊN)

1. Đăng nhập (Login)

Đặc tả: Quản trị viên nhập thông tin để truy cập hệ thống chung.

sequenceDiagram actor A as Admin/Chủ sạp participant WA as Web Admin/Chủ sạp participant API participant DB as Database A->>WA: Nhập Username & Password WA->>API: POST /api/Auth/Login API->>DB: Truy vấn thông tin dựa trên Username DB-->>API: Trả về dữ liệu (Mật khẩu băm + Phân quyền) API->>API: Xác thực Mật khẩu alt Sai thông tin API-->>WA: 401 Unauthorized WA-->>A: Báo lỗi "Đăng nhập thất bại" else Đúng thông tin API->>API: Tạo JWT Token / Cookie Authentication API-->>WA: 200 OK + Token WA->>WA: Lưu thông tin phiên đăng nhập WA-->>A: Chuyển hướng tới trang chủ (Dashboard) end

2. Đăng xuất chuẩn bảo mật (Dùng chung cho Admin & Chủ Sạp)

Đặc tả: Thu hồi quyền truy cập (Token Invalidation), ngắt kết nối Websocket và dọn dẹp bộ đệm cục bộ an toàn.

sequenceDiagram actor U as Người dùng (Admin / Merchant) participant WC as Web admin/Chủ sạp participant Hub as SignalR Hub participant API as Backend API participant DB as Database / Redis (Blacklist) U->>WC: Bấm nút "Đăng xuất" opt Nếu đang có kết nối Websocket WC->>Hub: Gửi lệnh ngắt kết nối (Disconnect) Hub-->>WC: Xác nhận đóng kết nối end WC->>API: POST /api/Auth/Logout (Gửi kèm Access Token) API->>DB: Đưa Token vào Blacklist (Vô hiệu hóa Refresh Token) DB-->>API: Ghi nhận thành công API-->>WC: 200 OK (Đã thu hồi phiên) WC->>WC: Clear LocalStorage / Cookies / SessionStorage WC-->>U: Chuyển hướng về trang Đăng Nhập

3. Xem thống kê báo cáo

Đặc tả: Thống kê số lượng theo Thời gian tìm kiếm (Start Date/End Date).

sequenceDiagram actor A as Admin participant WA as Web Admin participant API participant DB as Database A->>WA: Chọn Từ ngày... Đến ngày... WA->>API: GET /api/Stats?startDate=X&endDate=Y API->>DB: Đếm số lượng Sạp, User, Lượng Khách trong khoảng thời gian DB-->>API: Tổng hợp Object số liệu API-->>WA: Trả về JSON Stats WA-->>A: Vẽ biểu đồ / Hiển thị con số

4. Phân bổ Tour Tự động (Admin Auto-Assign)

Đặc tả: Admin khởi chạy tính năng quy hoạch lại bản đồ. Hệ thống sẽ tự động xáo trộn ngẫu nhiên các sạp hợp lệ và phân bổ đều vào 10 Lộ trình (Tours) theo thuật toán Round-Robin.

sequenceDiagram actor A as Admin participant WA as Web Admin participant API as ToursController participant DB as Database A->>WA: Bấm nút "Quy hoạch & Phân bổ Sạp" WA->>API: POST /api/Tours/auto-assign API->>DB: Truy vấn danh sách Lộ trình đang hoạt động (IsActive = true) DB-->>API: Trả về List 10 Tours API->>DB: Truy vấn danh sách Sạp hợp lệ (IsOpen = true & Còn hạn gói) DB-->>API: Trả về List Valid Stalls API->>API: Chạy thuật toán Shuffle (Xáo trộn ngẫu nhiên mảng Sạp) API->>API: Chạy thuật toán Round-Robin (Chia đều Sạp vào mảng Tour) API->>DB: Cập nhật cột TourID và SortOrder cho toàn bộ Sạp được phân bổ DB-->>API: SaveChangesAsync() - Hoàn tất lưu trữ API-->>WA: HTTP 200 OK + Message: "Đã quy hoạch lại toàn bộ Sạp" WA-->>A: Hiển thị thông báo thành công (Toast/SweetAlert) & Tải lại danh sách

5. Quản lý Danh sách Sạp trong Lộ trình (Tour Details)

Đặc tả: Luồng xử lý khi Admin thao tác cấu hình lộ trình chi tiết: Thêm một sạp tự do vào Tour, Thay đổi thứ tự (Lên/Xuống), và Gỡ sạp ra khỏi Tour.

sequenceDiagram actor A as Admin participant WA as Web Admin (DetailsModel) participant API as ToursController participant DB as Database Note over A, DB: Admin đang ở màn hình Chi tiết Lộ trình alt Trường hợp 1: Thêm Sạp vào Lộ trình A->>WA: Chọn sạp từ Dropdown & Bấm "Thêm vào lộ trình" WA->>API: PUT /api/Tours/{tourId}/AddStall/{stallId} API->>DB: Cập nhật TourID cho Sạp & Tính SortOrder mới (Max + 1) DB-->>API: Lưu dữ liệu thành công API-->>WA: HTTP 200 OK WA-->>A: Reload trang & Hiện thông báo Toast "Đã thêm sạp" else Trường hợp 2: Thay đổi thứ tự (Move Up/Down) A->>WA: Bấm nút Mũi tên (Lên hoặc Xuống) WA->>API: PUT /api/Tours/{tourId}/MoveStall/{stallId}?direction={dir} API->>DB: Tìm Sạp hiện tại & Sạp kề cận để Hoán đổi (Swap) SortOrder DB-->>API: Cập nhật lại toàn bộ thứ tự API-->>WA: HTTP 200 OK WA-->>A: Reload trang & Hiện thông báo "Đã cập nhật thứ tự" else Trường hợp 3: Gỡ Sạp khỏi Lộ trình A->>WA: Bấm nút "Xóa" & Đồng ý xác nhận (Confirm) WA->>API: PUT /api/Tours/{tourId}/RemoveStall/{stallId} API->>DB: Trả sạp về tự do (TourID = null & SortOrder = 0) DB-->>API: Cập nhật thành công API-->>WA: HTTP 200 OK WA-->>A: Reload trang & Hiện thông báo "Đã gỡ quán" end

6. Tìm kiếm, Lọc và Phân trang Sạp hàng

Đặc tả: Luồng xử lý đa điều kiện (Multi-criteria Filter) bao gồm tìm kiếm chuỗi, lọc theo thời gian, kiểm tra chéo dữ liệu Chủ sạp và phân trang kết quả.

sequenceDiagram actor A as Admin participant WA as Web Admin (IndexModel) participant API as Backend API A->>WA: Nhập điều kiện (SearchTerm, StartDate, EndDate, PageNumber) & Bấm "Lọc" par Gọi API đồng thời (Concurrent Tasks) WA->>API: GET api/Users/owners-summary WA->>API: GET api/Stalls end API-->>WA: Trả về Danh sách Chủ sạp (Owners) API-->>WA: Trả về Danh sách Toàn bộ sạp (allStalls) opt Có nhập từ khóa (SearchTerm != null) WA->>WA: Tìm list ID Chủ sạp (matchingOwnerIds) có Tên/SĐT khớp từ khóa WA->>WA: Lọc giữ lại Sạp có Tên chứa từ khóa HOẶC thuộc matchingOwnerIds end opt Nhập sai ngày (EndDate < StartDate) WA->>WA: Tự động hoán đổi (Swap) StartDate và EndDate WA->>WA: Lưu thông báo TempData["Error"] báo cho người dùng end opt Lọc theo Khoảng thời gian WA->>WA: Lọc giữ lại Sạp có UpdatedAt nằm trong khoảng StartDate - EndDate end WA->>WA: Tính TotalPages và cắt mảng dữ liệu (Skip & Take) theo PageNumber WA-->>A: Render Giao diện: Hiển thị Bảng danh sách Sạp đã lọc & Nút phân trang

7. Xóa Sạp (Delete Stall)

Đặc tả: Xóa bay 1 sạp hàng tại chức năng "Xóa ở các trang quản lý".

sequenceDiagram actor A as Admin participant MAP as Web Admin (Leaflet Map) participant API as StallsController participant DB as Database A->>MAP: Click vào Marker sạp (Đỏ hoặc Xanh) MAP-->>A: Hiện Popup thông tin sạp A->>MAP: Bấm nút "🗑️ Xóa sạp" MAP-->>A: Hiển thị Confirm: "Bạn có chắc chắn muốn xóa?" A->>MAP: Đồng ý (Confirm) MAP->>API: DELETE /api/Stalls/{id} API->>DB: Tìm Sạp & Cập nhật IsDeleted = true API->>DB: Gỡ sạp khỏi Tour & Cập nhật các nội dung liên quan (Soft Delete) DB-->>API: Success API-->>MAP: HTTP 200 OK MAP-->>A: Thông báo: "🗑️ Đã xóa sạp thành công!" MAP->>MAP: location.reload() (Tải lại bản đồ) MAP->>API: GET /api/Stalls/admin-map API-->>MAP: Trả về danh sách mới (không còn sạp vừa xóa) MAP-->>A: Cập nhật bản đồ (Marker biến mất)

PHẦN 2: WEB CHỦ SẠP (MERCHANT) VÀ THANH TOÁN TỰ ĐỘNG

sequenceDiagram actor M as Người Đăng Ký participant WM as Web Merchant participant API as PaymentsController participant Bank as SePay (Ngân hàng) participant DB as Database M->>WM: Nhập Info (Tên, SĐT, Tên Sạp, GPS) & Bấm Đăng Ký WM->>API: POST /api/Payments/register-new-merchant API->>API: Sinh mã đơn tạm (VD: DH123) & Lưu ổ cứng API-->>WM: Trả về URL QR Code WM-->>M: Hiển thị QR Code (2.000 VNĐ) M->>Bank: Quét mã QR & Chuyển khoản thành công Bank->>API: Gửi Webhook báo có tiền (Nội dung: DH123) Note over API, DB: --- Webhook xử lý Tự động --- API->>API: Đọc thông tin từ file tạm DH123 API->>DB: INSERT User mới (Role: StallOwner) API->>DB: INSERT Sạp mới (Gán OwnerId) API->>DB: INSERT Subscription (Dùng thử 30 ngày) DB-->>API: Lưu thành công M->>WM: Bấm "Tôi đã thanh toán" (Reload) WM-->>M: Chuyển hướng đăng nhập thành công
